3) Fork a repo: Clone

Clone repo you want to work with:

git clone (https address of the repo)

e.g.

git clone https://github.com/UCLU-techSoc/uclu-techsoc.github.io.git

Use ssh address if you have public key setup

You can also clone specific branch:

git clone -b (branch) (remote_repo)

e.g.

git clone -b my-branch https://git@github.com/username/myproject.git

You can find repo's address on github page:

4) Fork a repo: Configure remotes

When a repository is cloned, it has a default remote called:

origin

that points to your fork on GitHub, but not keeping updates from it. To keep track of the original repository, you need to add another remote named:

upstream

Go into your directory, we just created:

cd (folder you saved your project)

e.g.

cd MyProject

Assign the original repository to a remote called "upstream":

git remote add upstream (https address of the repo)

e.g.

git remote add upstream https://github.com/UCLU-techSoc/uclu-techsoc.github.io.git

Pulls in changes not present in your local repo, without modifying your files:

git fetch upstream

5) Push commits: Create branch

Before you will deal with any files update pn Git, the right way is to commit files to spererate branch first and then after approval of your team members merge it with master.

To create a branch and begin working in it, run these commands:

git branch (your branch name)

# Creates a new branch

git checkout (your branch name)

# Makes your branch active, so you can commit files to it.

6) Push commits: Add updated files. Commit. Push.

Now when you cloned and have changes to be made to repo, we need to push it to git.

Firstly add all changes by running:

git add .

# Add all files to commit

Then commit them:

git commit

After previous command you will have window for message summary of the update. Try to make it short and precise, then press Ctrl+X then Y to save message.

Now push all changes by running:

git push

Congratulations, your files should be commited, check this out on GitHub.

Other ways:

If you don't have specific branch as remote you can also push files:

git push (https address of the repo) (your branch or master)

7) Push commits: Merge

Once you're finished working on your branch and are ready to combine it back into the master branch, use merge:

git checkout master

# Makes "master" the active branch, so you can work on it

git merge mybranch

# Merges the commits from "mybranch" into "master"

git branch -d mybranch

# Deletes the "mybranch" branc
